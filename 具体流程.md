# LLM数学推理Agent — 具体开发流程（本地VSCode → AutoDL云端训练）

---

## 整体思路

```
本地 VSCode（无GPU）           Git仓库            AutoDL 云实例（RTX 4090）
    │                            │                      │
    ├─ 编写全部代码               │                      │
    ├─ CPU上做单元测试（mock LLM） │                      │
    ├─ 小规模逻辑验证              │                      │
    │                            │                      │
    └──── git push ────────►─────┼──── git pull ────────►│
                                 │                      ├─ pip install 环境
                                 │                      ├─ 运行训练脚本
                                 │                      ├─ 监控 wandb
                                 │                      └─ 下载 checkpoint
```

## Step 1：本地开发阶段（Phase 0，约1周，0费用）

在本地的 VSCode 上完成所有不需要 GPU 的工作。

### 环境准备

```bash
# 本地只安装开发需要的轻量依赖
pip install torch --cpu-only        # CPU版PyTorch，用于测试数据结构
pip install transformers datasets sympy pytest
pip install sentence-transformers   # embedding 测试
pip install black isort             # 代码格式化
# 不装 vllm / bitsandbytes（需要GPU）
```

### 开发策略——Mock 测试驱动

关键技巧：写一个 MockLLM 替代真实模型，用于本地测试 Agent 逻辑。

```python
class MockLLM:
    """模拟LLM输出，用于本地测试Agent逻辑"""
    def generate(self, prompt, **kwargs):
        # 返回预定义的推理文本，覆盖各种场景
        # 场景1: 包含有效工具调用
        # 场景2: 包含最终答案
        # 场景3: 格式错误
        return self._get_mock_response(prompt)

# 用 MockLLM 测试 ReActAgent 的完整循环
# 验证：动作解析、工具调用、停止条件、轨迹数据结构
# 完全不需要 GPU
```

### 本地可完成 vs 不可完成

| 可完成（本地） | 不可完成（需GPU） |
|----------------|-------------------|
| 项目结构、所有 .py 文件 | SFT/RL 训练 |

| MathEnvironment 交互逻辑 | 模型加载和推理 |
| ActionParser 解析逻辑 | 任何训练脚本的运行 |
| ReActAgent 循环逻辑（用mock） | |
| Memory MD 读写和 numpy 检索 | |
| MathJudge 答案判定 | |
| 数据加载和预处理 | |
| 所有单元测试（pytest） | |
| SFT训练数据构造 | |
| config.yaml 编写 | |

### 本地验证终点

- `pytest` 全绿
- 用 MockLLM 跑通 5 条完整推理轨迹，打印到控制台检查逻辑正确性

## Step 2：代码上云——AutoDL 实例配置

### 2.1 租用实例

1. 打开 [AutoDL](https://www.autodl.com)，选择 **RTX 4090 24G**（1.88元/时）
2. **镜像选择**（重点，省时间）：
   ```
   社区镜像 → 搜索带 PyTorch 2.1+ / CUDA 12.1+ 的镜像
   推荐：PyTorch 2.3.0 + CUDA 12.1 + Python 3.10（miniconda3）
   ```
3. 数据盘选择 **50GB** 足够：
   - 模型权重 ~15G
   - 数据集 ~1G
   - checkpoint/输出 ~5G
   - 环境依赖 ~5G

### 2.2 连接方式——VSCode Remote SSH

AutoDL 实例启动后在页面可看到连接信息：

```
SSH 地址: region-1.autodl.com
SSH 端口: 12345
用户名: root
密码: 从控制台复制
```

**本地 VSCode 配置**：

1. 安装 VSCode 插件：`Remote - SSH`
2. `Ctrl+Shift+P` → `Remote-SSH: Open SSH Configuration File`
3. 选择用户级配置文件（如 `~/.ssh/config`），添加：
   ```
   Host autodl-4090
       HostName region-1.autodl.com
       Port 12345
       User root
   ```
4. `Ctrl+Shift+P` → `Remote-SSH: Connect to Host` → `autodl-4090`
5. 输入密码，连接成功后在远程 VSCode 中打开终端

**效果**：在本地 VSCode 界面操作，代码和运行全在远程 GPU 上，无缝体验。

### 2.3 AutoDL 常用操作快捷入口

| 操作 | 方式 |
|------|------|
| 开机/关机 | AutoDL 网页控制台 |
| 上传文件 | JupyterLab 入口（实例页点击进入，支持拖拽上传） |
| 下载文件 | JupyterLab 文件浏览器 → 右键下载 |
| 查看GPU使用率 | 终端执行 `nvidia-smi` 或网页控制台面板 |

## Step 3：云端环境搭建（首次约30分钟）

连接上 AutoDL 后，在远程 VSCode 终端中执行：

```bash
# 1. 检查GPU是否正常
nvidia-smi

# 2. 进入数据盘目录（/root/autodl-tmp/ 是大容量数据盘）
cd /root/autodl-tmp/

# 3. 克隆代码仓库
git clone <你的仓库地址>
cd llm-math-agent

# 4. 安装依赖（镜像已有 torch，补装其余）
pip install transformers==4.44.0
pip install trl peft bitsandbytes datasets
pip install vllm
pip install wandb sympy scipy
pip install sentence-transformers langchain
pip install pytest              # 验证用的

# 5. 配置 HuggingFace 缓存到数据盘（避免占满系统盘）
export HF_HOME=/root/autodl-tmp/hf_cache
echo 'export HF_HOME=/root/autodl-tmp/hf_cache' >> ~/.bashrc

# 6. 验证环境
python -c "import torch; print(torch.cuda.is_available())"  # 应输出 True
python -c "import vllm; print('vllm OK')"
```

## Step 4：分阶段训练执行

### 每次上机的标准前奏

```bash
# 开机后先同步最新代码
cd /root/autodl-tmp/llm-math-agent
git pull

# 跑一遍测试确认代码正确
pytest tests/ -v
```

---

### Phase 1：SFT 训练（约3h GPU，约6元）

```bash
# 1. 下载基座模型（首次需要约15分钟，后续缓存）
huggingface-cli download Qwen/Qwen2.5-7B-Instruct

# 2. 运行 SFT 训练
python scripts/01_sft_train.py

# 训练过程关注：
# - 终端看 loss 是否下降
# - wandb.ai 网页看训练曲线
# - checkpoint 自动保存到 ./checkpoints/sft/

# 3. 训练完成后，快速验证格式合规率
python scripts/04_evaluate.py --model checkpoints/sft/ --dataset l1 --size 100
```

**判断标准**：格式合规率 > 90% → 进入 Phase 2；否则本地改 prompt/SFT数据 → `git push` → 云端 `git pull` → 重跑。

---

### Phase 2：RFT Baseline（约3h GPU，约5元）

```bash
# 运行RFT训练（2轮迭代）
python scripts/02_rft_train.py

# 训练完自动评估 MATH 验证集，结果打印到控制台
# 轨迹自动保存到 data/memory_store/
```

**判断标准**：MATH Acc 比 SFT 基线提升 > 3 个百分点 → 进入 Phase 3；否则本地分析失败轨迹 MD 文件 → 调参 → 重跑。

---

### Phase 3：GRPO 训练（约10-15h GPU，约28元）

```bash
# 关键：用 tmux 保持会话，防止 SSH 断开导致训练中断
tmux new -s grpo_train

# 进入项目目录
cd /root/autodl-tmp/llm-math-agent

# 运行 GRPO 训练
python scripts/03_grpo_train.py

# tmux 操作：
# Ctrl+B 然后 D  →  安全断开（训练继续后台跑）
# tmux attach -t grpo_train  →  重新连接查看状态
```

**训练中监控**（不需要保持 SSH 连接）：

```
通过 wandb 网页监控：
  1. 注册 https://wandb.ai（免费）
  2. 打开项目面板，找到 llm-math-agent
  3. 实时看曲线：
     - reward_mean：应持续上升
     - kl_divergence：应稳定在 < 0.1
     - accuracy（每50步评估一次）：应持续上升

  信号解读：
     - reward_mean 横盘超过50步 → 考虑调大学习率或减小KL系数
     - kl_divergence > 0.15 → 策略偏离太远，可能崩溃，调大KL系数
     - accuracy 剧烈波动 → 调低温度 / 增大group_size
```

**判断标准**：
- reward_mean 持续上升
- KL散度 < 0.1
- MATH Acc 比 RFT 再提升 > 3 个百分点 → Phase 4
- 不达标 → 本地分析 → 调超参 → 重跑

---

## Step 6：关机省钱守则

| 动作 | 说明 |
|------|------|
| 关机 | AutoDL 网页控制台点"关机"：**停止计费，数据保留** |
| 释放 | **数据全部清空！** 除非确定不要了 |
| 数据盘 | 如果买了独立数据盘，释放实例后数据盘仍保留，下次挂载即可 |

**不建议的做法**：
- 别开着机写代码（本地干）
- 别开着机分析结果（下载到本地看）
- 别开着机过夜（除非训练在跑）

**建议的节奏**：

```
早上   开机 → git pull → 跑训练
白天   wandb 网页监控（不需要连SSH）
晚上   训练结束 → 下载结果 → 关机
晚间   本地分析结果 + 改代码 → git push（准备明天）
周末   如果训练完整跑完 → 关机整个周末
```
